# Contributing to Peregrine

Спасибо за интерес к проекту! Мы рады любому вкладу.

## Начало работы

1. Форкните репозиторий
2. Создайте ветку: `git checkout -b feature/your-feature`
3. Внесите изменения
4. Убедитесь, что всё собирается и тесты проходят:

```bash
cargo build --workspace --all-features
cargo test --workspace --all-features
cargo clippy --workspace --all-features -- -D warnings
cargo fmt --all --check
```

5. Создайте Pull Request

## Соглашения

- **Код**: следуйте стилю, принятому в проекте (см. существующий код)
- **Коммиты**: пишите осмысленные сообщения на русском или английском
- **Issues**: перед началом работы над крупной фичей создайте issue для обсуждения
- **Тесты**: новые функции должны покрываться тестами

## Архитектурные заметки

- `peregrine-types` — единый источник правды для моделей данных. Не дублируйте типы.
- `peregrine-core` — чистая бизнес-логика, без зависимостей от Dioxus
- `api` — серверные функции для fullstack-режима
- `ui` — все UI-компоненты, общие для платформ
- Платформенные крейты (`web`, `desktop`, `mobile`) должны быть минимальными — только точка входа и роутинг
